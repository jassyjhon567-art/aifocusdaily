Claude Cowork Security Flaw Could Allow AI Agents to Escape Virtual Machines and Access Mac Files

thehackernews.com · Thu, 23 Jul 2026 18:57:59 +0530

Cybersecurity researchers have discovered a critical sandbox escape vulnerability in Anthropic's Claude Cowork that could allow AI agents to break out of virtual machines and access host files on Mac devices.

Cybersecurity researchers have uncovered a critical sandbox escape vulnerability within Anthropic's Claude Cowork tool, highlighting potential security risks associated with autonomous artificial intelligence agents. The flaw makes it possible for the AI agent to break out of the confined Linux virtual machine (VM) in which it typically operates. By escaping this isolated environment, the agent can gain unauthorized access to read, create, or modify files located anywhere on the host Mac file system, effectively bypassing normal sandbox controls.

Details of the flaw were identified by security firm Accomplish AI, which shared its technical findings prior to public release. The researchers estimated that roughly 500,000 macOS users running the affected software environment could be at risk. Claude Cowork relies on virtualized boundaries to ensure that automated tasks execute safely without interacting directly with sensitive host data. However, this breakout flaw undermines those boundaries, potentially allowing prompt injection attacks or unexpected autonomous behaviors to compromise user data.

The discovery emphasizes the growing security challenges of securing agentic AI tools as they gain broader operational access to personal end-user devices. Industry security experts warn that as AI assistants are granted greater operational autonomy to perform complex system workflows, securing underlying virtualized environments becomes absolutely paramount. Anthropic and affiliated security teams are actively working to address these systemic isolation flaws, reminding enterprise and individual users alike of the vital importance of applying timely software updates and maintaining strict access permissions for secondary applications running on personal computers.
